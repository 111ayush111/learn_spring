1. Java Classes & Objects 🎯

Bhai, sabse pehle yaad kar - Class ek blueprint hai jaise ghar ka naksha, aur Object us naksha se bana actual ghar hai.

Student.java
// Class = Blueprint (Naksha)
public class Student {
    
    // Properties (Variables) - Data store karne ke liye
    private Long id;
    private String name;
    private int age;
    
    // Constructor - Object banate waqt values set karne ke liye
    public Student() {}  // Default constructor (khali)
    
    public Student(Long id, String name, int age) {
        this.id = id;
        this.name = name;
        this.age = age;
    }
    
    // Getters & Setters - Data access karne ke liye
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}
Main.java
// Object = Actual Ghar (Blueprint se bana)
Student student1 = new Student(1L, "Rahul", 20);  // Object banaya!
Student student2 = new Student(2L, "Priya", 22);  // Dusra object!

System.out.println(student1.getName());  // Output: Rahul
System.out.println(student2.getName());  // Output: Priya
💡 Yaad Rakh
  • Class = Blueprint (naksha) - sirf definition hoti hai
  • Object = Real thing (actual ghar) - memory mein exist karta hai
  • new keyword se object banate hain
  • private variables sirf class ke andar access hote hain
  • public getters/setters se bahar se data access karte hain

Spring Boot Mein Kahan Use Hota Hai?

Spring Boot mein har cheez class hoti hai: Controller class, Service class, Repository class, Entity class - sab classes hain! Toh class/object concept clear hona must hai.

2. Inheritance & Interfaces 🔗

Inheritance (Baap-Beta Concept)

Inheritance matlab - baap ki property beta inherit kar lega. Java mein extends keyword use karte hain.

Animal.java (Parent Class)
public class Animal {
    protected String name;  // protected = baap aur dono ke liye
    
    public void eat() {
        System.out.println(name + " kha raha hai");
    }
}
Dog.java (Child Class)
// Dog extends Animal = Dog ne Animal ki properties le li
public class Dog extends Animal {
    
    public void bark() {
        System.out.println(name + " bhaunk raha hai!");
    }
}
Main.java
Dog dog = new Dog();
dog.name = "Tommy";
dog.eat();    // Animal se mila - Output: Tommy kha raha hai
dog.bark();   // Dog ka apna - Output: Tommy bhaunk raha hai!

Interface (Contract/Agreement)

Interface ek contract hai - jo class interface implement karegi, usko saare methods define karne padenge. Java mein implements keyword use karte hain.

AnimalActions.java (Interface)
public interface AnimalActions {
    void makeSound();   // Sirf declaration, NO body!
    void move();      // Sirf declaration, NO body!
}
Cat.java (Implements Interface)
public class Cat implements AnimalActions {
    
    @Override
    public void makeSound() {
        System.out.println("Meow Meow!");
    }
    
    @Override
    public void move() {
        System.out.println("Cat chal rahi hai");
    }
}
🧠 Spring Boot Mein Interface Ka Kya Kaam?

Spring Boot mein JpaRepository ek interface hai! Jab tu extends JpaRepository karta hai, toh automatically CRUD methods mil jaate hain - tuje khud likhne ki zaroorat nahi! Yehi Spring Boot ka magic hai - Interface inheritance se kaam asaan ho jata hai.

3. Collections (List, Map) 📚

Collections data store karne ke tarike hain. Spring Boot mein data ko handle karne ke liye collections ka bahut use hota hai (jaise student list, product list, etc.)

List (Ordered Collection - Duplicate allowed)

ListExample.java
import java.util.ArrayList;
import java.util.List;

// List = Ordered items (jaise line mein khade students)
List<String> names = new ArrayList<>();

// Items add karna
names.add("Rahul");
names.add("Priya");
names.add("Amit");

// Access karna (index se - 0 se start hota hai)
System.out.println(names.get(0));  // Rahul

// Loop karke sab print karna
for (String name : names) {
    System.out.println(name);
}

Map (Key-Value Pair - Dictionary jaise)

MapExample.java
import java.util.HashMap;
import java.util.Map;

// Map = Key-Value pairs (jaise phonebook - Name -> Number)
Map<String, Integer> ages = new HashMap<>();

// Key-Value put karna
ages.put("Rahul", 20);
ages.put("Priya", 22);
ages.put("Amit", 21);

// Key se value nikalna
System.out.println(ages.get("Rahul"));  // 20
💡 Spring Boot Mein Kahan Use Hota Hai?
  • List - Database se saare records nikalne ke liye (e.g., List<Student> students)
  • Map - JSON response banane ke liye (e.g., error messages key-value format mein)

4. Exception Handling ⚠️

Exception = Error jo program run karte waqt aata hai. Agar handle nahi kiya toh program crash ho jayega!

ExceptionExample.java
public class ExceptionExample {
    public static void main(String[] args) {
        
        try {
            // Risky code jo error de sakta hai
            int result = 10 / 0;  // Divide by zero! 💥
        } 
        catch (ArithmeticException e) {
            // Error aaya toh yahan handle karo
            System.out.println("Bhai, 0 se divide nahi kar sakte!");
        }
        finally {
            // Ye hamesha chalega chahe error aaye ya na aaye
            System.out.println("Ye toh chalega hi!");
        }
        
        // Program aage chalega - crash nahi hoga!
        System.out.println("Program successfully chal raha hai!");
    }
}
Keyword Kaam
try Risky code dalne ke liye jo error de sakta hai
catch Error pakadne aur handle karne ke liye
finally Hamesha chalega - cleanup ke liye (optional)
throw Khud se error throw karne ke liye
📌 Spring Boot Mein Exception Handling
Spring Boot mein @ExceptionHandler annotation se exceptions handle karte hain. Isse agar API mein error aaye toh user ko accha error message mil jata hai instead of ugly stack trace.

5. Java Annotations (@) 🏷️

Annotation matlab - code ke upar likha hua special marker jo compiler ya framework ko batata hai ki iske saath kya karna hai. Spring Boot pure annotations pe based hai!

Annotations Kya Hote Hain?

Soch le - tu apne doston ko party ke liye invite karta hai. Har invite card pe @ likhke kuch special instruction deta hai jaise @DressCode Formal ya @Venue BanquetHall. Waise hi Java mein @ se kuch special instruction dete hain.

AnnotationExample.java
// @Override = Bata raha hai ki ye method parent class se override ho raha hai
@Override
public void makeSound() {
    System.out.println("Meow!");
}

// @Deprecated = Ye method purana ho gaya hai, mat use karo
@Deprecated
public void oldMethod() {
    // Purana code
}

// @SuppressWarnings = Compiler warnings ko suppress karne ke liye
@SuppressWarnings("unchecked")
public void someMethod() {
    // Code with warnings
}
⚠️ Important
Annotations = Magic nahi, Instructions hain! Annotation likhne se behind the scenes kaam hota hai. Spring Boot ke annotations se framework ko pata chalta hai ki kaunsa class kya kaam karega. Ye sab reflection mechanism se hota hai.

Annotations Ke Parts

Annotation Type Kya Hota Hai? Example
Marker Annotation Sirf mark karta hai, koi data nahi @Override
Single-Value Annotation Ek value deta hai @RequestMapping("/api")
Multi-Value Annotation Kai values deta hai @GetMapping(path="/users", produces="application/json")
Meta Annotation Annotation pe annotation @Target, @Retention

Spring Boot Kya Hai? 🌱

Spring Boot Java mein applications banane ka framework hai jo fast, easy aur production-ready apps banane mein madad karta hai. Isme already bahut saari cheezein configured hoti hain - tujhe bas apna business logic likhna hai!

Spring Boot Ke Fayde

  • Auto Configuration - Bahut saari cheezein automatically set up hoti hain
  • Standalone - Server (Tomcat) already included hai, alag se install nahi karna
  • Opinionated - Best practices by default follow karta hai
  • Microservices - Choti-choti services banane ke liye perfect hai
  • Production Ready - Health checks, metrics sab built-in hain
📌 Spring vs Spring Boot

Spring = Framework with lots of configuration (XML likhna padta tha)
Spring Boot = Spring ka wrapper jo configuration kam kar deta hai (Annotations se kaam chal jata hai)

Annotations Deep Dive 🔍

Bhai, ye section sabse important hai! Annotations Spring Boot ka dil hain. Har annotation ko detail mein samajh lega toh kabhi problem nahi hogi.

@SpringBootApplication

Kya hai? Ye 3 annotations ka combination hai - main class ke upar lagta hai

Kyun lagate hain? Spring Boot ko batata hai ki "Bhai, ye main class hai, yahan se shuru karna!"

Tere Repo Ka Code
@SpringBootApplication  // = @Configuration + @EnableAutoConfiguration + @ComponentScan
public class ProductlistApplication {
    public static void main(String[] args) {
        SpringApplication.run(ProductlistApplication.class, args);
    }
}
Andar Ka KaamMatlab
@Configuration"Is class mein configuration hai"
@EnableAutoConfiguration"Automatically configure kar de jo chahiye"
@ComponentScan"Is package aur sub-packages mein components dhoondh"

@RestController

Kya hai? Class ko REST Controller banata hai - HTTP requests handle karne wala

Kyun lagate hain? Is class mein jo methods honge wo API endpoints ban jayenge

Combination: @Controller + @ResponseBody

@RestController  // "Bhai, ye class API requests handle karegi"
@RequestMapping("/api/students")  // Base path for all APIs
public class StudentController {
    
    @GetMapping("/")  // GET /api/students/ = Sab students lao
    public List<Student> getAllStudents() {
        return studentService.findAll();
    }
}

@GetMapping, @PostMapping, @PutMapping, @DeleteMapping

Kya hai? HTTP methods ko Java methods se map karne ke liye

AnnotationHTTP MethodKaamExample
@GetMapping GET Data read/get karne ke liye @GetMapping("/{id}")
@PostMapping POST Naya data create karne ke liye @PostMapping("/")
@PutMapping PUT Poora data update karne ke liye @PutMapping("/{id}")
@DeleteMapping DELETE Data delete karne ke liye @DeleteMapping("/{id}")
@PatchMapping PATCH Partial update ke liye @PatchMapping("/{id}") 
@RestController
@RequestMapping("/api/students")
public class StudentController {

    // GET http://localhost:8080/api/students → Sab students
    @GetMapping
    public List<Student> getAll() { ... }

    // GET http://localhost:8080/api/students/1 → Student with id=1
    @GetMapping("/{id}")
    public Student getById(@PathVariable Long id) { ... }

    // POST http://localhost:8080/api/students → Naya student create
    @PostMapping
    public Student create(@RequestBody Student student) { ... }

    // PUT http://localhost:8080/api/students/1 → Student update
    @PutMapping("/{id}")
    public Student update(@PathVariable Long id, @RequestBody Student student) { ... }

    // DELETE http://localhost:8080/api/students/1 → Student delete
    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) { ... }
}

@PathVariable

Kya hai? URL path se variable value nikalne ke liye

Example: URL /students/5 mein se 5 ko id variable mein dalna

// URL: /api/students/5 → id = 5
@GetMapping("/{id}")
public Student getStudent(@PathVariable Long id) {
    // id ki value 5 hogi!
    return studentService.findById(id);
}

// Multiple path variables bhi le sakte hain
@GetMapping("/{studentId}/courses/{courseId}")
public Course getCourse(
        @PathVariable Long studentId,
        @PathVariable Long courseId) {
    // Dono values mil jayengi!
}

@RequestBody

Kya hai? HTTP request ke body se JSON data ko Java object mein convert karta hai

Example: Client ne {"name":"Rahul","age":20} bheja → Student object ban jayega

// Client bhejega: {"name": "Rahul", "age": 20, "email": "rahul@test.com"}
@PostMapping
public Student createStudent(@RequestBody Student student) {
    /* 
       Spring Boot automatically JSON → Student object convert kar dega!
       student.getName() = "Rahul"
       student.getAge() = 20
    */
    return studentService.save(student);
}
💡 @RequestBody Ka Magic

@RequestBody lagane se Spring Boot Jackson library use karke automatically JSON ko Java object mein convert kar deta hai. Tujhe khud parse nahi karna padta!

@RequestParam

Kya hai? URL ke query parameters se value nikalne ke liye

Example: URL /students?name=Rahul mein se name ki value nikalna

// URL: /api/students/search?name=Rahul → name = "Rahul"
@GetMapping("/search")
public List<Student> searchByName(@RequestParam String name) {
    return studentService.findByName(name);
}

// Optional parameter bhi bana sakte hain
@GetMapping("/filter")
public List<Student> filter(
        @RequestParam(required = false) String name,
        @RequestParam(required = false, defaultValue = "18") int minAge) {
    // name optional hai, minAge default 18 hogi
}
📌 @PathVariable vs @RequestParam

@PathVariable = URL path se value → /students/5
@RequestParam = URL query se value → /students?name=Rahul

@Autowired

Kya hai? Spring ko batata hai ki is field/method mein dependency inject karni hai

Simple bhasha mein: Spring automatically object bana ke de dega, tujhe new nahi karna!

@RestController
public class StudentController {
    
    @Autowired  // "Spring bhai, StudentService ka object bana ke yahan daal de!"
    private StudentService studentService;
    
    // Ab tu studentService use kar sakta hai - new nahi karna pada!
}
💡 Modern Spring Boot Mein (Recommended)

Ab @Autowired lagane ki zaroorat nahi hai - Constructor Injection use karo: 

@RestController
public class StudentController {
    
    private final StudentService studentService;
    
    // Constructor se Spring automatically inject kar dega - @Autowired nahi chahiye!
    public StudentController(StudentService studentService) {
        this.studentService = studentService;
    }
}

@Entity, @Table, @Id, @GeneratedValue

Kya hai? Java class ko Database Table se map karne ke liye (JPA)

@Entity  // "Ye class ek database table hai"
@Table(name = "students")  // Table ka naam "students" hoga
public class Student {
    
    @Id  // "Ye primary key hai"
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // Auto increment
    private Long id;
    
    @Column(name = "student_name", nullable = false)  // Column mapping
    private String name;
    
    @Column
    private int age;
    
    // Getters & Setters
}
AnnotationKaam
@EntityClass ko database table mark karta hai
@Table(name="xyz")Table ka naam set karta hai
@IdPrimary key column mark karta hai
@GeneratedValueID auto-generate karta hai
@ColumnColumn properties set karta hai

@Repository

Kya hai? Class ko Data Access Layer mark karta hai - database se baat karne wala

Spring ise automatically scan karta hai aur @Autowired ke time inject kar deta hai

@Repository  // "Ye class database se baat karegi"
public interface StudentRepository extends JpaRepository<Student, Long> {
    // JpaRepository extend karne se CRUD methods automatically mil jaate hain!
    // save(), findAll(), findById(), deleteById() - sab ready!
}

@Service

Kya hai? Class ko Business Logic Layer mark karta hai

Controller se data aata hai → Service mein process hota hai → Repository mein jata hai

@Service  // "Ye class business logic handle karegi"
public class StudentService {
    
    private final StudentRepository studentRepository;
    
    public StudentService(StudentRepository studentRepository) {
        this.studentRepository = studentRepository;
    }
    
    public List<Student> findAll() {
        return studentRepository.findAll();
    }
    
    public Student save(Student student) {
        // Business logic - validation, processing, etc.
        return studentRepository.save(student);
    }
}

REST API Kya Hai? 🌐

REST API (Representational State Transfer) ek tarika hai jisse do applications internet ke zariye baat kar sakti hain. Soch le - teri Spring Boot app server hai aur browser/phone app client hai. REST API unke beech ka raasta hai.

Real Life Example

Restaurant mein soch: Tu client hai, waiter API hai, aur kitchen server hai. Tu waiter ko order deta hai (request) → waiter kitchen mein jata hai → khana le ke aata hai (response). Waiter = API jo tuje kitchen ke saare internals nahi batata, bas order leke response deta hai!

📌 REST API Ke Rules
  • Stateless - Har request independent hoti hai, server ko pehli request ki yaad nahi rehti
  • Resource-based - Har cheez ek resource hai (jaise /students, /courses)
  • JSON/XML - Data JSON format mein exchange hota hai
  • HTTP Methods - GET, POST, PUT, DELETE use hote hain

JPA & Hibernate 🗄️

JPA (Java Persistence API) ek standard hai jo Java objects ko database tables se map karne ka tarika batata hai. Hibernate is JPA ka implementation hai jo actually ye kaam karta hai.

📌 Simple Samajh

JPA = Rules/Standard (interface)
Hibernate = Rules follow karne wala (implementation)

Tu JPA annotations use karta hai → Hibernate unhe padhta hai → Database mein SQL queries chalata hai

Kaam Kaise Karta Hai?

  1. Tu Java class banata hai aur @Entity lagata hai
  2. Class ke variables pe @Column lagata hai
  3. Hibernate automatically database table bana deta hai
  4. Tu Java objects se kaam karta hai - SQL likhne ki zaroorat nahi!

Entity Class Banana 👤

Entity class ek Java class hai jo database table se map hoti hai. Har entity object ek table row represent karta hai.

Student.java (Entity Class)
package com.bitsandbytes.studentapi.model;

import jakarta.persistence.*;  // JPA annotations

@Entity                     // Ye class ek database table hai
@Table(name = "students")   // Table ka naam
public class Student {

    @Id                                             // Primary key
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // Auto increment
    private Long id;

    @Column(name = "student_name", nullable = false, length = 100)
    private String name;

    @Column(nullable = false)
    private int age;

    @Column(unique = true)  // Duplicate email nahi hone chahiye
    private String email;

    // ===== CONSTRUCTORS =====
    
    // Default constructor (JPA ke liye zaroori hai!)
    public Student() {}

    // Parameterized constructor
    public Student(String name, int age, String email) {
        this.name = name;
        this.age = age;
        this.email = email;
    }

    // ===== GETTERS & SETTERS =====
    /* Ye zaroori hain kyunki:
       - JPA values set/get karta hai in methods se
       - JSON serialization/deserialization ke liye chahiye
    */
    
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }

    // ===== toString() =====
    @Override
    public String toString() {
        return "Student{" +
                "id=" + id +
                ", name='" + name + '\'' +
                ", age=" + age +
                ", email='" + email + '\'' +
                '}';
    }
}
Entity Annotations Explained
AnnotationAttributesKya Karega?
@Entity - Class ko database table mark karega
@Table name Table ka naam customize karega
@Id - Primary key column mark karega
@GeneratedValue strategy ID auto-generate karega (IDENTITY = auto-increment)
@Column name, nullable, length, unique Column properties set karega
⚠️ Important Rules
  • Entity class mein default constructor hona chahiye (JPA use karta hai)
  • @Id field honi chahiye - ye primary key hai
  • Getters aur Setters hone chahiye - JPA inka use karta hai
  • Class final nahi honi chahiye

package com.bitsandbytes.studentapi.service;

import com.bitsandbytes.studentapi.model.Student;
import com.bitsandbytes.studentapi.repository.StudentRepository;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Optional;

@Service
public class StudentService {

    private final StudentRepository studentRepository;

    // Constructor Injection - Spring automatically object de dega
    public StudentService(StudentRepository studentRepository) {
        this.studentRepository = studentRepository;
    }

    // ===== CREATE =====
    public Student createStudent(Student student) {
        // Validation
        if (student.getName() == null || student.getName().trim().isEmpty()) {
            throw new IllegalArgumentException("Name is required!");
        }
        if (student.getAge() <= 0) {
            throw new IllegalArgumentException("Age must be positive!");
        }
        return studentRepository.save(student);
    }

    // ===== READ ALL =====
    public List<Student> getAllStudents() {
        return studentRepository.findAll();
    }

    // ===== READ ONE =====
    public Optional<Student> getStudentById(Long id) {
        return studentRepository.findById(id);
    }

    // ===== UPDATE =====
    public Student updateStudent(Long id, Student studentDetails) {
        Student student = studentRepository.findById(id)
                .orElseThrow(() -> new RuntimeException("Student not found with id: " + id));

        student.setName(studentDetails.getName());
        student.setAge(studentDetails.getAge());
        student.setEmail(studentDetails.getEmail());

        return studentRepository.save(student);
    }

    // ===== DELETE =====
    public void deleteStudent(Long id) {
        Student student = studentRepository.findById(id)
                .orElseThrow(() -> new RuntimeException("Student not found with id: " + id));
        studentRepository.delete(student);
    }

    // ===== SEARCH =====
    public List<Student> searchByName(String name) {
        return studentRepository.findByNameContainingIgnoreCase(name);
    }
}

// Jab error aayegi toh ye format mein response jayega
public class ErrorResponse {
    private int status;
    private String message;
    private LocalDateTime timestamp;

    public ErrorResponse(int status, String message) {
        this.status = status;
        this.message = message;
        this.timestamp = LocalDateTime.now();
    }

    // Getters
    public int getStatus() { return status; }
    public String getMessage() { return message; }
    public LocalDateTime getTimestamp() { return timestamp; }
}

GlobalExceptionHandler Class

exception/GlobalExceptionHandler.java
package com.bitsandbytes.studentapi.exception;

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

/*
 * @RestControllerAdvice = Ye class puri application ke exceptions handle karegi
 * Har controller mein alag se try-catch nahi likhna padega!
 */
@RestControllerAdvice
public class GlobalExceptionHandler {

    // Jab bhi IllegalArgumentException aayegi, ye method chalega
    @ExceptionHandler(IllegalArgumentException.class)
    public ResponseEntity<ErrorResponse> handleIllegalArgument(IllegalArgumentException ex) {
        ErrorResponse error = new ErrorResponse(HttpStatus.BAD_REQUEST.value(), ex.getMessage());
        return new ResponseEntity<>(error, HttpStatus.BAD_REQUEST);  // 400
    }

    // Jab bhi RuntimeException aayegi (jaise "Student not found")
    @ExceptionHandler(RuntimeException.class)
    public ResponseEntity<ErrorResponse> handleRuntimeException(RuntimeException ex) {
        ErrorResponse error = new ErrorResponse(HttpStatus.NOT_FOUND.value(), ex.getMessage());
        return new ResponseEntity<>(error, HttpStatus.NOT_FOUND);  // 404
    }

    // General Exception - koi aur unexpected error
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleGeneralException(Exception ex) {
        ErrorResponse error = new ErrorResponse(
            HttpStatus.INTERNAL_SERVER_ERROR.value(),
            "Something went wrong: " + ex.getMessage()
        );
        return new ResponseEntity<>(error, HttpStatus.INTERNAL_SERVER_ERROR);  // 500
    }
}
🧠 Exception Handler Ka Magic

@RestControllerAdvice lagane se ye class global exception handler ban jata hai. Ab kisi bhi controller mein error aaye toh automatically ye pakad lega aur accha JSON response dega. Har controller mein alag se try-catch likhne ki zaroorat nahi!
